第4篇-linux下安装软件的3种方式

前言

首次接触到linux,发现这种操作系统中和windows上很不相同。传统的windows操作系统下，只需要下载好安装程序，一路点击Next,就可以实现程序的安装，最复杂的也不过是要配置一下环境变量而已。而在linux操作系统中，安装方式多种多样。这篇博客主要是通过介绍windowns和linux下安装软件的原理出发，对比两者异同。最后，介绍linux下安装linux软件的三种方法。

软件安装的机制

参考文章：

  1. linux应用程序的安装和卸载
  2. windows和linux安装方式的比较

在操作系统环境中，程序主要分为两种。一种是普通的安装软件，另一种是供其他软件使用的依赖库。大多数的程序都是要利用系统中所安装的依赖库。在安装软件的过程中，每个软件都有对应的配置文件。所以，一个软件的安装可以简单的概括为链接依赖库和配置对应的配置文件。

  • windows下的软件安装过程 ：

windows下系统的安装过程，都是有一个安装程序来完成的。这个安装文件的格式是一个exe格式的，最后安装好之后，也会生成一个exe文件格式的软件。安装背后的具体操作如下：

（1）在Program Files 文件夹下建立程序文件夹。

（2）复制相关的依赖库到程序文件夹下或者系统文件夹下。windows下的依赖库文件格式为.dll

（3）复制可执行文件到程序文件夹下，比如对应的exe文件。

（4）复制配置文件到程序文件夹或者系统文件夹下。windows下的配置文件格式为.ini

（5）把启动配置或者程序的依赖配置放到注册表中。

（6）如果有自定义服务程序，注册并启动服务程序。

从上面的过程中，我们可以总结出，软件的安装的过程有点想生物里面的细胞分裂。无非就是将软件包需要的文件复制到本地，然后更改一下配置文件，链接一下所需要的依赖库，最后就可以运行了。

  • linux下系统的安装过程

    （1）复制相关类库

    （2）复制可执行程序

    （3）配置到相应的目录

    （4）根据需要选择性配置和开启服务

linux下软件安装的三种方式。

在linux操作系统中，主要有三种方式来安装软件。分别是源码安装，使用软件包管理工具dpkg安装,
以及使用apt-get软件包管理包来安装

1. 源码安装

源码安装的步骤如下

(1)从官网下载源码，并且解压缩到指定的文件夹

(2)进入软件目录运行命令./configure,生成配置文件

(3)make ，编译，将源码编译成对应的二进制文件。

(4)make install 安装。此过程对应上面所叙述的复制类库等操作。

(5)make uninstall 卸载软件。

2. 使用软件包管理工具dpkg来安装

使用软件包管理工具来安装软件类似于windows中的安装方式。这些软件包都是已经编译好了的
二进制格式的安装包格式。常见的软件包格式有一下两种

软件包格式 应用的场景
rmp Red Hat linux的软件包格式
deb Debian Linux的软件包格式，是一种二进制格式的软件包安装格式。全称为debian binary.由于Ubutu是基于debian的，所以在Ubuntu的操作系统中，也可是使用这种格式的安装包

在使用软件包的安装方式中，用到的工具是dpkg的debian的包管理器，全称是debian package.对于dpkg的具体用法，可以用man dpkg 来查询。
但是dpkg现在应用的比较少，主要是因为有一下缺点：

1. 不能主动在镜像网站来获取软件包

2. 安装软件包时，不能自动的安装相关依赖包

对于dpkg的包操作，主要有一下几种：

操作命令 含义
dpkg -i <package> 利用下载好了的包来安装程序
dpkg -r <package> 移除安装好的程序
dpkg -p <package> 移除安装好的程序以及配置文件
dpkg -l <package> 列出所有安装包的清单
dpkg -info 查看包中的信息

3. 使用在线管理工具apt-get来安装

apt-get是一个在线的智能管理工具，可以用来安装，卸载和升级软件。它的实现原理是，读取镜像站点的地址，自动查找对应的软件信息，自动的安装相关的依赖程序，从而安装程序。在这种安装方法中，经常要做一件事情，那就是更改镜像源。因为linux默认的镜像源服务器在国外，下载软件的过程十分的缓慢，一般都需要更改镜像源，比较好的源有阿里的和网易的这些。更改镜像源的文件地址在/etc/apt/sources.list

对于这种安装方式，常用的命令主要有一下几种：

命令 含义
apt-get update 更新镜像源的地址
apt-get upgrade 自动升级软件包到最新的版本
apt-get install <package> 安装软件
apt-get -f install 修复依赖包的关系
apt-get remove <package> 卸载软件
apt-get remove –purge <package> 卸载完全，包括配置文件等信息
apt-get install -d 只下载（deb）软件包，不安装。使用apt-get 下载的软件包默认的存放路径是：/var/cache/apt/archives

综上所述，apt-get install 来安装软件的基本工作步骤如下

1. 扫描本地软件包列表（apt-get update 刷新软件包列表)

2. 进行软件包依赖关系检查(自动进行)

3. 从软件包指定的apt源中下载软件包以及相应的依赖库

4. 解压缩软件包，并完成软件的安装和配置

第3篇-linux的文件目录及路径

前言

在linux操作系统中，有着一切皆文件的说法，对于文件路径来说，是经常会用到的。所以，在学习的过程中，不可避免的对这些知识点有了疑惑，于是整理一番，有了接下来的这篇文章。

文件路径的表示方法及其含义

在操作系统中，表示路径有两种方式，一种是绝对路径，一种是相对路径。

  1. 绝对路径：写法一定是从根目录“/”开始的。
  2. 相对路径：根据当前所处的目录来表示目标目录。

在操作系统中，每个目录都有两个特殊的目录，“.”和“..”目录。

  • “.”目录表示当前目录
  • “..”目录表示当前目录的上一个目录。

用绝对路径表示根目录下的etc目录的目录dir下的file文件：/etc/dir/file
若我们当前在file目录下，那用相对路径表示etc目录为：../etc

linux操作系统中的文件分类

linux操作系统中的文件结构如下：

文件夹名 存放的文件
/bin 用以存储二进制可执行命令文件
/sbin 许多系统命令的存储位置，/usr/sbin/中也包括了许多命令。
/home 普通用户的默认目录，在该目录下，每个用户拥有一个以用户名命名的文件夹。
/dev 存储设备文件，包括计算机的所有外部设备，如硬盘、是、键盘、鼠标等。
/boot 存放Ubuntu内核和系统启动文件。
/etc 存放配置文件
/usr 包括与系统用户直接有关的文件和目录
/opt 作为可选文件和程序的存放目录，一般自己安装的软件存放在该目录下

第2篇-转-linux系统在用户与用户组的概念

前言：

本来想总结一下用户和用户组之间的关系。但是，后来发现了这篇博客，写的真的是非常的全面，于是就决定将其转载到自己的笔记中，在这里对原文的排版进行了一些改动，以便让自己对知识的掌握层次更加清晰。

在此声明，本文的原文内容转载地址：linux下查看所有用户及所有用户组

Linux 是一个多用户、多任务的操作系统。我们应该了解单用户多任务和多用户多任务的概念。

Linux 的单用户多任务

单用户多任务：比如我们以beinan 登录系统，进入系统后，我要打开gedit 来写文档，但在写文档的过程中，我感觉少点音乐，所以又打开xmms 来点音乐；当然听点音乐还不行，MSN 还得打开，想知道几个弟兄现在正在做什么，这样一样，我在用beinan 用户登录时，执行了gedit 、xmms以及msn等，当然还有输入法fcitx ；这样说来就有点简单了，一个beinan用户，为了完成工作，执行了几个任务；当然beinan这个用户，其它的人还能以远程登录过来，也能做其它的工作。

Linux 的多用户、多任务

有时可能是很多用户同时用同一个系统，但并不所有的用户都一定都要做同一件事，所以这就有多用户多任务之说；举个例子，比如LinuxSir.Org 服务器，上面有FTP 用户、系统管理员、web 用户、常规普通用户等，在同一时刻，可能有的弟兄正在访问论坛；有的可能在上传软件包管理子站，比如luma 或Yuking 兄在管理他们的主页系统和FTP ；在与此同时，可能还会有系统管理员在维护系统；浏览主页的用的是nobody 用户，大家都用同一个，而上传软件包用的是FTP用户；管理员的对系统的维护或查看，可能用的是普通帐号或超级权限root帐号；不同用户所具有的权限也不同，要完成不同的任务得需要不同的用户，也可以说不同的用户，可能完成的工作也不一样。

值得注意的是：多用户多任务并不是大家同时挤到一接在一台机器的的键盘和显示器前来操作机器，多用户可能通过远程登录来进行，比如对服务器的远程控制，只要有用户权限任何人都是可以上去操作的。

用户的角色区分

用户在系统中是分角色的，在Linux 系统中，由于角色不同，权限和所完成的任务也不同；值得注意的是用户的角色是通过UID和识别的，特别是UID；在系统管理中，系统管理员一定要坚守UID 唯一的特性.

  1. root 用户：系统唯一，是真实的，可以登录系统，可以操作系统任何文件和命令，拥有最高权限；
  2. 虚拟用户：这类用户也被称之为伪用户或假用户，与真实用户区分开来，这类用户不具有登录系统的能力，但却是系统运行不可缺少的用户，比如bin、daemon、adm、ftp、mail等；这类用户都系统自身拥有的，而非后来添加的，当然我们也可以添加虚拟用户；
  3. 普通真实用户：这类用户能登录系统，但只能操作自己家目录的内容；权限有限；这类用户都是系统管理员自行添加。

多用户操作系统的安全

多用户系统从事实来说对系统管理更为方便。从安全角度来说，多用户管理的系统更为安全，比如beinan用户下的某个文件不想让其它用户看到，只是设置一下文件的权限，只有beinan一个用户可读可写可编辑就行了，这样一来只有beinan一个用户可以对其私有文件进行操作，Linux 在多用户下表现最佳，Linux能很好的保护每个用户的安全，但我们也得学会Linux 才是，再安全的系统，如果没有安全意识的管理员或管理技术，这样的系统也不是安全的。

从服务器角度来说，多用户的下的系统安全性也是最为重要的，我们常用的Windows 操作系统，它在系纺权限管理的能力只能说是一般般，根本没有没有办法和Linux或Unix 类系统相比；

理解用户和用户组的概念

  1. 用户（user）的概念；

通过前面对Linux 多用户的理解，我们明白Linux 是真正意义上的多用户操作系统，所以我们能在Linux系统中建若干用户（user）。比如我们的同事想用我的计算机，但我不想让他用我的用户名登录，因为我的用户名下有不想让别人看到的资料和信息（也就是隐私内容）这时我就可以给他建一个新的用户名，让他用我所开的用户名去折腾，这从计算机安全角度来说是符合操作规则的.

当然用户（user）的概念理解还不仅仅于此，在Linux系统中还有一些用户是用来完成特定任务的，比如nobody和ftp 等，我们访问LinuxSir.Org 的网页程序，就是nobody用户；我们匿名访问ftp 时，会用到用户ftp或nobody ；如果您想了解Linux系统的一些帐号，请查看 /etc/passwd .

2、用户组（group）的概念

用户组（group）就是具有相同特征的用户（user）的集合体；比如有时我们要让多个用户具有相同的权限，比如查看、修改某一文件或执行某个命令，这时我们需要用户组，我们把用户都定义到同一用户组，我们通过修改文件或目录的权限，让用户组具有一定的操作权限，这样用户组下的用户对该文件或目录都具有相同的权限，这是我们通过定义组和修改文件的权限来实现的；

举例：我们为了让一些用户有权限查看某一文档，比如是一个时间表，而编写时间表的人要具有读写执行的权限，我们想让一些用户知道这个时间表的内容，而不让他们修改，所以我们可以把这些用户都划到一个组，然后来修改这个文件的权限，让用户组可读，这样用户组下面的每个用户都是可读的.

3、用户和用户组的对应关系

用户和用户组的对应关系是：一对一、多对一、一对多或多对多.

一对一 某个用户可以是某个组的唯一成员
多对一 多个用户可以是某个唯一的组的成员，不归属其它用户组；比如beinan和linuxsir两个用户只归属于beinan用户组

第1篇-如何增删Linux用户？

前言:

  在学习python的过程中，不可避免的要接触到linux。在这个过程之中，慢慢的发现，其实，知识之间的连接是十分广泛的。对于一个完全的新手来说，通过一天天的总结，一点点的积累，我相信最后一定可以成为一个linuxer。于是开始从自己的学习经历出发，边学习，边总结，于是，我决定写这一系列的笔记。在这个过程中，我用到的linux系统是Ubuntu16.04 LTS.

linux中增加用户的操作

  在linux系统下，创建用户的命令有两种。
   命令1：useradd username
  这条命令只能够在centos系统下使用。在Ubuntu系统下使用这条命令，虽然会创建用户名，但是不能登录到桌面系统。只能在命令行系统中进行登录。
   命令2：adduser username(推荐+Ubuntu系统)
  这条命令可以在Ubuntu系统下创建新的用户名，并且在/home目录下，创建用户目录，从而可以用户登录。

linux 用户中删除用户的操作

  常见的用户删除命令中，最长用到的是userdel username即可.
操作过程在可能出现的情况：
   报错:username is currently used by process Id

解决方案：

方案1:更改配置文件以删除用户
用法：vipw [选项]
选项：

-g, --group                  编辑 group 数据库 
-h, --help                    显示此帮助信息并推出 
-p, --passwd                  编辑 passwd 数据库 
-q, --quiet                  安静模式 
-R, --root CHROOT_DIR        chroot 到的目录 
-s, --shadow                  编辑 shadow 或 gshadow 数据库
***

具体步骤
  第一条命令：vipw,选择一个编辑器之后，删除之前创建用户的那一行即可。
  第二条命令：vipw -s 打开shadow文件，并删除用户所属组。

方案2:更改配置文件以删除用户
  重启电脑，进入其中一个用户。在这个户用环境中，以root用户权限来使用userdel -r username即可.

linux中用户权限的切换以及设置用户密码

用户权限的设置:
  linux操作系统之中，有些操作必须要用到root用户权限。用到root用户权限有两种方法。第一种：sudo + 相关命令。第二种， su 输入密码，切换到超级用户。
退出超级用户的操作:
  方法一：输入 su 普通用户名  方法二：exit

是日也， 天朗气清， 惠风和畅。 仰观宇宙之大， 俯察品类之盛， 所以游目骋怀， 足以极视听之娱， 信可乐也。

夫人之相与， 俯仰一世。 或取诸怀抱， 悟言一室之内； 或因寄所托， 放浪形骸之外。 虽趣舍万殊， 静躁不同， 当其欣于所遇， 暂得于己， 快然自足， 不知老之将至； 及其所之既倦， 情随事迁， 感慨系之矣。 向之所欣， 俯仰之间， 已为陈迹， 犹不能不以之兴怀， 况修短随化， 终期于尽！ 古人云： “死生亦大矣。” 岂不痛哉！

每览昔人兴感之由， 若合一契， 未尝不临文嗟悼， 不能喻之于怀。 固知一死生为虚诞， 齐彭殇为妄作。 后之视今， 亦犹今之视昔， 悲夫！ 故列叙时人， 录其所述， 虽世殊事异， 所以兴怀， 其致一也。 后之览者， 亦将有感于斯文。

劝学

君子曰：学不可以已。青，取之于蓝，而青于蓝；冰，水为之，而寒于水。木直中绳。輮以为轮，其曲中规。虽有槁暴，不复挺者，輮使之然也。故木受绳则直，金就砺则利，君子博学而日参省乎己，则知明而行无过矣。

吾尝终日而思矣，不如须臾之所学也；吾尝跂而望矣，不如登高之博见也。登高而招，臂非加长也，而见者远；顺风而呼，声非加疾也，而闻者彰。假舆马者，非利足也，而致千里；假舟楫者，非能水也，而绝江河，君子生非异也，善假于物也。

积土成山，风雨兴焉；积水成渊，蛟龙生焉；积善成德，而神明自得，圣心备焉。故不积跬步，无以至千里；不积小流，无以成江海。骐骥一跃，不能十步；驽马十驾，功在不舍。锲而舍之，朽木不折；锲而不舍，金石可镂。蚓无爪牙之利，筋骨之强，上食埃土，下饮黄泉，用心一也。蟹六跪而二螯，非蛇鳝之穴无可寄托者，用心躁也。
